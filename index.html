AWS CloudFormation Template DSL

Alexis King <lexi.lambda@gmail.com>

 #lang aws/cloudformation/template
  package: aws-cloudformation-template-lib

NOTE: This library is unstable; compatibility will not be maintained.

Amazon Web Services provides a service called CloudFormation, a tool for managing the creation, update, and removal of entire stacks of services at once, which is useful for complex deployments that use many different AWS services for a single application. These stacks are defined using declarative templates, which are then interpreted by CloudFormation.

For certain scenarios, CloudFormation is an invaluable tool, but templates can quickly become unwieldy and difficult to maintain. They can be written in JSON or YAML, but support features that simple configuration formats are ill-equipped to handle, such as including variable references and even calls to predefined functions. Additionally, the resource types can be quite complex, and it can be easy to forget what resources expect or allow certain options.

This library provides a statically typed DSL for constructing CloudFormation templates. Running a module written in the aws/cloudformation/template language will produce a JSON template on stdout that can be directly provided to AWS.

1 Example

Here’s a simple demonstration of a template that uses some of the features that this library provides:

"template.rkt"

#lang aws/cloudformation/template
 
(defparam network-service-subnets : (List String)
  #:description "The subnets to which the load balancer attaches.")
 
(defparam cluster : Resource
  #:description "The ECS cluster for container deployment")
 
(defparam task-definition : Resource
  #:description "The ECS task definition containing the container to run")
 
(defresource load-balancer
  (aws:elastic-load-balancing:load-balancer
   #:listeners [{ #:instance-port 8080
                  #:instance-protocol "HTTP"
                  #:load-balancer-port 80
                  #:protocol "HTTP"}]
   #:subnets network-service-subnets))
 
(defresource service
  (aws:ecs:service
   #:cluster cluster
   #:role "ecsServiceRole"
   #:load-balancers [{ #:container-name "http-server"
                       #:container-port 8080
                       #:load-balancer-name load-balancer}]
   #:desired-count 2
   #:task-definition task-definition))

2 Reference

2.1 Base Types

type

String

type

Number

type

Boolean

The types of raw data. String literals are surrounded with " double quotes ", numbers are integers or decimals, and booleans are written as #t or #f.

type

(List type)

syntax

[elem-expr ...]

The type of lists and list literals, which correspond to arrays in AWS CloudFormation templates. A list literal can be written by surrounding elements with [ square brackets ]. All elements of a list must be the same type.

> ["hello" "world"]

'("hello" "world")

> [{ #:x 0 #:y 0}  { #:x 10 #:y 5}]

'(#hash((X . 0) (Y . 0)) #hash((X . 10) (Y . 5)))

type

(Dict type)

syntax

(dict kw elem-expr ... ...)

The type of dictionaries and dictionary literals, which correspond to arbitrary JSON objects in AWS CloudFormation templates.

> (dict #:x "hello"
        #:y "world")

'#hash((x . "hello") (y . "world"))

type

(Record record-spec ...)

 
record-spec = required-spec
  | optional-spec
     
required-spec = #:<key> val-type
     
optional-spec = [#:<key> val-type]

syntax

{ #:<key> val-expr ... ... }

The type of records, which are heterogenous sets of key/value pairs. All keys must be keywords, but values can be of any type. Key/type pairs wrapped with [ square brackets ] are optional.

Record literals are written as keyword/expression pairs surrounded by { curly braces }.

> {}

'#hash()

> { #:x 0 #:y 0}

'#hash((X . 0) (Y . 0))

> { #:foo { #:bar { #:baz #f}}}

'#hash((Foo . #hash((Bar . #hash((Baz . #f))))))

type

Resource

The type of all AWS resources.

2.2 Definition Forms

syntax

(defparam id : type
  #:description description-str)
Declares a parameter named id as an input to a template with the type type and description description-str, which must be a string literal. The parameter can be referenced elsewhere in the template.

syntax

(defresource id resource-expr)

 
  resource-expr : Resource
Declares a resource to be created by this template with the name id and described by resource-expr. The resource can be referenced elsewhere in the template.

3 AWS Intrinsic Functions and Pseudo Parameters

value

aws:region : String

Returns a string representing the AWS Region in which the encompassing resource is being created, such as us-west-2.

value

aws:stack-name : String

Returns the name of the stack.

procedure

(fn:or x y) → Boolean

  x : Boolean
  y : Boolean
Returns #t if either x or y evaluate to #t, otherwise returns #f.

procedure

(fn:join [#:separator separator] strs) → String

  separator : String = ""
  strs : (List String)
Concatenates a set of strings into a single string, separated by separator.

procedure

(fn:if condition x y) → a

  condition : Boolean
  x : a
  y : a
Returns x if condition evaluates to #t, otherwise returns y.

procedure

(fn:equal? x y) → Boolean

  x : a
  y : a
Returns #t if x and y are equivalent values, otherwise returns #f.
